Copy `.env.example` to `.env.local` and configure:

```bash
# Required for chat feature (OpenAI-compatible API)
LLM_API_URL=https://your-litellm-server.com/v1/chat/completions
LLM_API_KEY=your-api-key
LLM_MODEL=qwen3-vl:Cloud
LLM_TEMPERATURE=0.7
LLM_MAX_TOKENS=500

# Required for admin node editor
NEXT_PUBLIC_ADMIN_PASSWORD=your-secure-password

# Optional rate limiting overrides
RATE_LIMIT_RPM=10
MAX_MESSAGE_LENGTH=2000
MAX_CONVERSATION_LENGTH=20
REQUEST_TIMEOUT=30000
```

## Architecture

### Data Flow

Infrastructure data flows through a pipeline:

1. `src/data/infrastructure.json` — flat array of `InfraItem` nodes with `parent` refs
2. `buildInfraTree()` (`src/lib/buildTree.ts`) — converts flat array into `InfraTreeNode[]` tree
3. `treeToReactFlow()` (`src/lib/treeToFlow.ts`) — converts tree into React Flow `nodes[]` + `edges[]`
4. `applyView()` (`src/lib/viewFilter.ts`) — filters visible nodes/edges based on active view
5. React Flow renders the final diagram in `DiagramContent` (`src/app/components/diagram/LabDiagram.tsx`)

Cross-connections (e.g., SSH links) are defined separately in `src/data/connections.json` and merged during step 3. Node status (up/down/off/unknown) comes from `src/data/status.json`. View configurations (which nodes to show) are in `src/data/views.json`.

### Key Data Types (`src/lib/infraTypes.ts`)

- `InfraItem` — flat node with optional `parent` field (ID string) to form the tree
- `InfraTreeNode` — extends `InfraItem` with `children: InfraTreeNode[]`

### Main Component: `LabDiagram`

`src/app/components/diagram/LabDiagram.tsx` is the core component. It's wrapped in `ReactFlowProvider` and the inner `DiagramContent` manages all state:

- **Topology state** (`currentTopology`) — the in-memory `InfraItem[]`, starts from `infrastructure.json` but can be mutated by the admin editor or cleared for "new canvas" mode
- **Layout persistence** — node positions saved to `localStorage` via `src/lib/layoutStorage.ts` (key: `lab-layouts`, max 5 per non-admin user)
- **Admin mode** — tracked in `localStorage` (`isAdmin` key); admin can permanently save topology to disk via `/api/save-infrastructure`
- **Custom mode** (`isCustomMode`) — when user creates a blank canvas; `onResetDiagram` restores original

### AI Chat

`ChatWithInfra` (`src/app/components/chat/ChatWithInfra.tsx`) sends messages to `/api/chat` (Next.js route). The chat API proxies to any OpenAI-compatible LLM. Context optimization in `src/lib/chatHelpers.ts`:

- By default, sends only relevant context based on keyword detection in the query
- If query contains full-topology keywords ("all", "topology", "completo", etc.), sends the full JSON
- Jailbreak detection patterns block off-topic queries client-side before hitting the API
- System prompt is built per-message with the selected context injected

### API Routes

- `POST /api/chat` — LLM proxy with IP-based rate limiting (in-memory, resets on server restart), message length validation, and 30s timeout
- `POST /api/save-infrastructure` — writes to `src/data/infrastructure.json` on disk; requires `NEXT_PUBLIC_ADMIN_PASSWORD` in request body

### Views System

Views in `src/data/views.json` define which nodes are visible. Each view has an `include` array of node IDs. When `expandChildren: true`, all descendants of included nodes are also shown. The `"full"` view uses `include: ["*"]` to show everything.

### Node Icons & Colors

`treeToFlow.ts` assigns Lucide React icons and hex colors based on node `type` and `id`. To add a new node type with a custom icon/color, update the `iconColors` record and `getIcon()` function in that file.

### Export Options

From the Toolbar, users can export as:
- JSON (topology data)
- PNG (via `html-to-image` on the React Flow element)
- Mermaid diagram (`.mmd` format, generated by `src/lib/mermaidExport.ts`)
